# Guia Pratico de Curriculos ATS

Este guia resume boas praticas para criar curriculos legiveis por ATS (Applicant Tracking Systems) sem perder qualidade para recrutadores humanos.

## Objetivo

- Maximizar parse correto no ATS
- Facilitar leitura rapida por recrutadores
- Reduzir risco de rejeicao por formato

## Principios de ouro

- Use layout simples em uma coluna
- Mantenha consistencia em todo o documento
- Priorize clareza, dados concretos e resultados
- Evite elementos visuais que quebram parse

## Estrutura recomendada (estilo "Harvard-like" + ATS)

Ordem sugerida:

1. Nome e contato (email, telefone, cidade/UF, LinkedIn/GitHub com URL completa)
2. Resumo profissional curto (2-4 linhas)
3. Experiencia
4. Educacao
5. Skills tecnicas
6. Certificacoes (quando relevante)
7. Projetos (quando relevante para a vaga)

Observacoes:

- Use nomes de secao padrao: `Experience`, `Education`, `Skills`, `Certifications`, `Projects`
- Ordem cronologica reversa (mais recente primeiro)
- Evite nomes criativos de secao que o ATS nao reconheca

## Datas: mito x pratica segura

## Mito comum

- "ATS nao le Jan 2026"

## Realidade

- Muitos ATS modernos leem `Jan 2026` ou `January 2026`
- O maior problema costuma ser inconsistencia ou formatos ambiguos

## Formato mais seguro para padronizar

- Preferir `MM/YYYY` em todo o curriculo
- Exemplo: `01/2026-03/2028`

Evite:

- `Summer 2026`
- `Jun '26`
- `5/19-8/20`
- Misturar varios formatos no mesmo documento

## Conteudo que melhora score ATS

- Inclua palavras-chave da vaga de forma natural
- Use verbos de acao no inicio dos bullets
- Mostre impacto com numeros
  - Exemplo: "Reduzi custo operacional em 18%"
- Relacione skill com evidencia de uso na experiencia
- Adapte o curriculo para cada vaga (nao use versao totalmente generica)

## Formato e tipografia (seguro para ATS)

- Fonte simples: Arial, Calibri, Times New Roman
- Tamanho legivel: 10-12 (corpo), 12-16 (titulos)
- Espacamento consistente
- PDF final limpo (sem corrupcao de texto)

Evite:

- Tabelas, colunas complexas, caixas de texto
- Icones, fotos, graficos, elementos decorativos
- Header/footer com informacoes importantes
- Emojis e caracteres especiais ornamentais

## Checklist rapido antes de exportar

- [ ] Todas as datas no mesmo formato
- [ ] Secoes com nomes padrao
- [ ] Bullets com verbo + contexto + resultado
- [ ] Pelo menos 3-5 palavras-chave da vaga aparecem na experiencia
- [ ] Sem tabelas, sem icones, sem imagens
- [ ] Contato no topo, fora de header/footer
- [ ] Ortografia revisada
- [ ] PDF final com texto selecionavel (nao imagem)

## Padrao recomendado para o Resuna

Para equilibrar conforto de UX e compatibilidade ATS:

- Entrada de data amigavel no formulario (mes + ano)
- Armazenamento canonico interno: `YYYY-MM`
- Exportacao ATS padrao: `MM/YYYY`
- Validador de formato bloqueando datas ambiguas
- Template padrao unico "Harvard ATS"
